Summary of Usability Testing Results

Respondents completed predefined tasks on the BoardEase prototype and were evaluated using the System Usability Scale (SUS). The SUS consists of 10 questions rated from 1 (Strongly Disagree) to 5 (Strongly Agree). The complete SUS scores (Q1–Q10) will be updated once full data is finalized. Based on the available data, overall user experience was generally positive with some notable areas for improvement.

Heuristic Issues Identified

Key Issues Identified

The following usability issues were identified through the Heuristic Evaluation conducted by the group leader (Ernesto P. Casasis Jr.) using Jakob Nielsen's 10 Usability Heuristics, and corroborated by usability testing observations.


1

Lack of Loading Feedback During Booking

When a user clicks "Book Now" or "Confirm" on the calendar, the app lacks a clear loading animation or spinner, leaving users uncertain whether their action was registered.

High Severity H1: Visibility of System Status
2

Technical Jargon in Interface Labels

Some interface headers and buttons use generic or technical design jargon instead of local student-friendly terms like "Boarding Houses for Rent", "Dormitories", or "Bedspace".

Medium Severity H2: Match Between System and the Real World
3

No Undo/Edit Option for Sent Messages

On the Chat/Inbox page, once a student sends a message to a landlord, there is no way to undo, edit, or delete that message, causing frustration during testing.

High Severity H3: User Control and Freedom
4

Inconsistent UI Components Across Screens

Back arrow buttons and confirmation blocks differ in visual styling across screens — some use solid background headers while others use a minimal approach, breaking platform consistency.

Medium Severity H4: Consistency and Standards
5

Price Range Filter Accepts Invalid Input

The "Price Range" filter uses a free-text input box that accepts letters, which can easily cause errors. No validation or constraint prevents invalid data entry.

Medium Severity H5: Error Prevention
6

Active Filters Disappear After Closing Filter Menu

Once the filter menu is closed, selected criteria (e.g., Aircon, Wifi) are no longer visible on the main screen, forcing users to recall their applied settings from memory.

Medium Severity H6: Recognition Rather Than Recall
7

No Quick Unsave Option in Save List

The "Save List" view has no direct mechanism (e.g., heart toggle or remove button) to quickly unsave an item. Users must navigate into the individual listing to undo a saved preference.

Low Severity H7: Flexibility and Efficiency of Use
8

Oversized Welcome Banner Reduces Listing Visibility

The "Welcome to BoardEase" banner on the home screen is disproportionately large, pushing the actual boarding house listings down and creating visual noise.

Low Severity H8: Aesthetic and Minimalist Design
9

No Error Message for Unavailable Date Selection

When a user attempts to select past or unavailable dates on the date selection screen, the system blocks the action but provides no explanation, leaving users confused.

High Severity H9: Help Users Recognize, Diagnose, and Recover from Errors
10

No Help Center or FAQs in Settings

The Settings screen completely lacks any link to a Help Center, FAQs, or customer support documentation, leaving users with no guidance for booking-related inquiries.

Medium Severity H10: Help and Documentation

Discussion of Findings

Heuristic Evaluation Findings

The heuristic evaluation conducted by the group leader revealed consistent and clearly defined usability issues across the BoardEase prototype. The most critical violations — rated as High Severity — were identified in Heuristic 1 (Visibility of System Status), Heuristic 3 (User Control and Freedom), and Heuristic 9 (Error Recovery). These three heuristics directly affect the user's ability to understand system behavior and recover from mistakes, both of which are fundamental to a positive user experience.

Usability Testing Findings

During the usability testing sessions with 40 BSED Science respondents, the majority of users were able to navigate the basic features of BoardEase, particularly the search and browsing functions. However, several participants encountered confusion during the booking flow — specifically when no loading feedback was present — and during filter use, where applied criteria were not persistently displayed. Users with prior experience using similar apps (52.5%) generally performed tasks with fewer hesitations, while first-time users required more time to locate key functions.

Relationship Between Evaluation Methods

The issues identified through heuristic evaluation were largely confirmed by the behavioral observations gathered during usability testing. Issues such as the missing loading indicator (H1) and the disappearing filter chips (H6) were directly observed as points of confusion among multiple respondents. This convergence across both evaluation methods validates the reliability of the identified issues and underscores the importance of addressing the high-severity findings before advancing the prototype to a higher-fidelity version.

Recommendations for Improving the Prototype

Based on both the Heuristic Evaluation and Usability Testing, the following design improvements are recommended:


🔄

Add Loading Indicators for Booking Actions

Implement a circular spinner or "Processing Booking…" overlay after any booking submission to keep users informed of background system activity.

🏷️

Use Student-Friendly Terminology

Revise interface labels to use locally familiar terms such as "Boarding Houses for Rent", "Dormitories", and "Bedspace" instead of generic technical jargon.

↩️

Enable Message Edit/Delete on Long Press

Add a long-press gesture on chat bubbles that allows users to delete or edit recently sent messages, improving control and freedom in the messaging flow.

🎨

Standardize UI Components

Create a unified design system for headers, back buttons, and confirmation blocks to ensure visual consistency across all screens.

🔢

Replace Price Range Text Box with Slider

Implement a numeric range slider for the Price Range filter to prevent invalid input and make the interaction faster and more intuitive.

🏷️

Persist Active Filters as Chips

Display active filter selections as dismissible chips below the search bar so users can always see and manage their applied criteria without reopening the filter menu.

💾

Add Quick Unsave Action to Save List Cards

Integrate a heart icon toggle or remove button directly on each card in the Save List view to eliminate the need to navigate into individual listings to unsave.

📐

Reduce Welcome Banner Size

Redesign the home screen banner to be more compact, giving greater visual priority to the actual boarding house listings and reducing visual clutter.

⚠️

Add Clear Error Messages for Date Selection

Show a user-friendly toast message (e.g., "This date is unavailable. Please choose another date.") whenever a user attempts to select blocked or past dates.

📖

Add Help & FAQs to Settings

Include a dedicated "Help & FAQs" section in the Settings screen to give users immediate access to support documentation and booking guidance.
